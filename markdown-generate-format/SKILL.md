# Markdown 文件格式规范

## Skill 目标

在生成 Markdown 文档时，确保遵循统一的格式规范，包括：

- 标题编号体系
- 严格换行规则

## 换行规则

### 各级标题下的换行

标题和正文之间必须有空行分隔。

**正确示例：**

```markdown
# 一级标题

一级标题下面的正文

## 二级标题

二级标题下面的正文

### 三级标题

三级标题下面的正文
```

**错误示例：**

```markdown
# 一级标题
一级标题下面的正文

## 二级标题
二级标题下面的正文
```

### 严格换行

根据 Markdown 标准，单个换行符在预览模式下不生效。段落之间必须用空行分隔。

**正确示例：**

```markdown
这是第一个段落。

这是第二个段落。
```

**错误示例：**

```markdown
这是第一个段落。
这是第二个段落。
```

预览时上述错误示例会变成："这是第一个段落。这是第二个段落。"

### 其他正文内容的换行

无序列表、有序列表、代码块、引用 `>` 等正文内容与段落之间也需要空行分隔。

**正确示例：**

```markdown
这是第一个段落。

- 无序列表。项目一
- 无序列表。项目二

这是第二个段落。

1. 有序列表。项目一
2. 有序列表。项目二

> 这是引用段落
```

**注意**：无序列表和有序列表的各个项之间不需要空行。

**错误示例：**

```markdown
- 无序列表。项目一

- 无序列表。项目二

- 无序列表。项目三
```

## 标题规则

### 标题编号体系

| 级别 | 语法      | 编号格式      | 示例               |
| -- | ------- | --------- | ---------------- |
| 一级 | `#`     | 无编号       | `# 标题`           |
| 二级 | `##`    | 中文数字 + 顿号 | `## 一、标题`        |
| 三级 | `###`   | 数字 + 顿号   | `### 1、标题`       |
| 四级 | `####`  | 数字.数字     | `#### 1.1 标题`    |
| 五级 | `#####` | 数字.数字.数字  | `##### 1.1.1 标题` |

### 一级标题

一级标题 `#` 不使用标号，一级标题代表整篇文章的标题。

```markdown
# 一级标题
```

### 二级标题

二级标题 `##` 使用中文数字，汉字后面跟顿号。

- 格式：`## 二级标题内容`
- 示例：`## 一、概述`、`## 二、实施步骤`
- 注意：顿号和标题内容之间不需要空格

### 三级标题

三级标题 `###` 使用数字，数字后面跟顿号。

- 格式：`### 1、标题内容`
- 示例：`### 1、项目背景`、`### 2、需求分析`
- 注意：顿号和标题内容之间不需要空格
- 二级标题下面的三级标题序号独立，如 `一、` 下的三级标题从 `1、` 开始

### 四级标题

四级标题 `####` 使用数字加点号格式。

- 格式：`#### 1.1 标题内容`
- 点前面的数字表示所属的三级标题
- 示例：`#### 1.1 需求概述`（属于三级标题 `1、`）、`#### 2.1 技术方案`（属于三级标题 `2、`）
- 序号和标题内容之间有空格

### 五级标题

五级标题 `#####` 使用数字加点号格式。

- 格式：`##### 1.1.1 标题内容`
- 第一个点前面的数字继承自三级标题，第二个点前面的数字继承自四级标题
- 示例：`##### 1.1.1 功能需求`（属于四级标题 `1.1`）
- 序号和标题内容之间有空格

## 执行要求

生成 Markdown 文档时：

1. 先规划标题结构，确定各级标题的编号
2. 标题和正文之间必须有空行
3. 段落之间必须有空行
4. 列表项之间不需要空行（同一列表的项之间紧凑排列）
