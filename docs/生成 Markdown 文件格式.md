# 生成 Markdown 文件格式

## 一、应用场景

在需要生成 Markdown 文档的场景。比如生成 Plan 文档、Spec 文档、Readme 等 Markdown 格式的文档。

## 二、规则

### 1、换行

#### 1.1 各级标题下的换行

##### 1.1.1 示例

```
# 一级标题

一级标题下面的正文

## 二级标题

二级标题下面的正文

## 三级标题

三级标题下面的正文
```

**注意**：上面的示例只展示了一、二、三级标题，其余级别的标题也要遵循。

##### 1.1.2 反例

```
# 一级标题
一级标题下面的正文

## 二级标题
二级标题下面的正文

## 三级标题
三级标题下面的正文
```

#### 1.2 严格换行

在生成的 Markdown 文件中要执行严格换行。根据 Markdown 标准，单个换行符在预览模式下不再生效（无法实际换行）。

##### 1.2.1 普通段落

```
这是第一个段落。

这是第二个段落。
```

像上面这样，每一个段落和每一个段落之间用换行分隔。

下面是一个反例。

```
这是第一个段落。
这是第二个段落。
```

上面这种反例，在预览时实际会变成下面这样：

```
这是第一个段落。这是第二个段落。
```

因为两个段落之间没有换行，所以在严格换行场景，进行预览时，两个段落实际上是一段。

##### 1.2.2 其他正文内容

另外，不只是普通的文字段落。

像无序列表、有序列表、代码块、引用 `>` 等正文内容都要换行。

```
这是第一个段落。

- 无序列表。项目一
- 无序列表。项目二
- 无序列表。项目三

这是第二个段落。

1. 有序列表。项目一
2. 有序列表。项目二
3. 有序列表。项目三

这是第三个段落。

> 这是引用段落，段落一
> 
> 这是引用段落，段落二
```

另外，无序列表和有序列表的列表中的各个项不需要换行：

```
- 无序列表。项目一

- 无序列表。项目二

- 无序列表。项目三
```

```
1. 有序列表。项目一

2. 有序列表。项目二

3. 有序列表。项目三
```

像上面这两个就是反例。

### 2、标题

各个级别的标题有一个标题号，就跟当前文档一样。

#### 2.1 一级标题

一级标题 `#` 不使用标号，一级标题代表整片文章的标题。

#### 2.2 二级标题

二级标题 `##` 使用中文数字，例如一、二、三、四、五、六、七、八、九、十、十一、十二

汉字后面要跟一个顿号，例如 `一、`

顿号和后面的标题内容之间不需要空格，例如 `一、这是一级标题`，一、顿号、这是一级标题之间没有空格。

#### 2.3 三级标题

三级标题 `###` 使用数字，例如 1、2、3

数字后跟着顿号，例如 `1、`、`2、`、`3、`

顿号和后面的标题内容之间不需要空格，例如 `1、这是三级标题`，1、顿号、这是三级标题之间没有空格。

二级标题下面的三级标题的序号是独立的，比如二级标题 `一、` 下面的三级标题从 `1、` 开始，接下来是 `2、`，二级标题 `二、` 下面的三级标题还是从 `1、` 开始，接下来是 `2、`。

#### 2.4 四级标题

四级标题 `####` 使用数字

数字之间例如 `1.1`、`1.2`、`1.3`

点前面的 `1` 表示该四级标题是 `1、` 这个三级标题下面的。如果四级标题为 `2.1`、`2.2`、`2.3`，点前面的 `2` 表示该四级标题是 `2、` 这个三级标题下面的。

四级标题中，点前面的数字继承自三级标题，点后面的数字为四级标题的序号。

四级标题序号和后面的标题内容之间使用空格分隔，例如 `1.1 这是四级标题`，`1.1` 和 `这是四级标题` 之间有一个空格。

#### 2.5 五级标题

五级标题 `#####` 使用数字

数字之间例如 `1.1.1`、`1.1.2`、`1.1.3`

第一个点前面的数字为三级标题，第二个点前面的数字为四级标题，第二个点后面的数字为五级标题

五级标题中，第一个点前面的数字继承自三级标题，第二个点前面的数字继承自四级标题，第二个点后面的数字为五级标题序号。

五级标题序号和后面的标题内容之间使用空格分隔，例如 `1.1.1 这是五级标题`，`1.1` 和 `这是五级标题` 之间有一个空格。
