# Java 代码中的注释规范

本 skill 只约束需要写 `/** */` 的注释场景：

- 成员变量注释
- 类注释
- 方法注释

不约束：

- 行注释 `//`
- 普通块注释 `/* */`

完整规范请阅读 [references/Java 代码中的注释规范.md](references/Java%20代码中的注释规范.md)。

## 一、执行要点

### 1、段落规则（`<p>`）

在成员变量注释、类注释和方法注释中：

- 每一个段落前必须使用 `<p>` 标签
- 段落之间用空行分隔（即 `*` 行之间保留一行空行）

**标准结构示例：**

```java
/**
 * <p>段落一
 *
 * <p>段落二
 */
```

### 2、列表规则（`<pre></pre>`）

当注释中出现无序列表或有序列表时：

- 列表内容必须放在 `<pre></pre>` 中
- 列表项以 `- ` 开头（与规范示例保持一致）

**示例：**

```java
/**
 * <p>无序列表
 * <pre>
 * - 第一条
 * - 第二条
 * </pre>
 */
```

### 3、块标签规则（`@author`）

生成 Java 类注释时，块标签必须包含：

- `@author  z2huo`

**示例：**

```java
/**
 * <p>
 *
 * @author  z2huo
 */
```

## 二、输出要求（强约束）

当你需要为 Java 代码生成/修改 `/** */` 注释时：

1. 只要是“段落”，就以 `* <p>` 开头
2. 只要出现“列表”，就用 `<pre></pre>` 包裹，并使用 `- ` 作为列表项前缀
3. 若生成/修改的是“类注释”，必须包含 `@author  z2huo`
4. 不要把该规范套用到 `//` 或 `/* */` 上
