# Java 代码中的注释规范

本 skill 只约束需要写 `/** */` 的注释场景：

- 成员变量注释
- 类注释
- 方法注释

不约束：

- 行注释 `//`
- 普通块注释 `/* */`

完整规范请阅读 [references/Java 代码中的注释规范.md](references/Java%20代码中的注释规范.md)。

## 一、执行要点

### 1、段落规则（`<p>`）

在成员变量注释、类注释和方法注释中：

- 第一个段落前不使用 `<p>` 标签
- 从第二个段落开始，每一个段落前必须使用 `<p>` 标签
- 段落之间用空行分隔（即 `*` 行之间保留一行空行）

**标准结构示例：**

```java
/**
 * 段落一
 *
 * <p>段落二
 */
```

### 2、列表规则（`<pre></pre>` / `<ul></ul>` / `<ol></ol>`）

当注释中出现无序列表或有序列表时：

- 列表内容可使用 `<pre></pre>`、`<ul></ul>` 或 `<ol></ol>`
- 使用 `<pre></pre>` 时，列表项以 `- ` 开头（与规范示例保持一致）
- 使用 `<ul></ul>` 或 `<ol></ol>` 时，列表项使用 `<li>`

**示例（`<pre></pre>`）：**

```java
/**
 * <p>无序列表
 * <pre>
 * - 第一条
 * - 第二条
 * </pre>
 */
```

**示例（`<ul></ul>`）：**

```java
/**
 * <p>无序列表
 * <ul>
 *     <li>第一条
 *     <li>第二条
 * </ul>
 */
```

**示例（`<ol></ol>`）：**

```java
/**
 * <p>有序列表
 * <ol>
 *     <li>第一条
 *     <li>第二条
 * </ol>
 */
```

### 3、块标签规则（`@author`）

生成 Java 类注释时，块标签必须包含：

- `@author z2huo`

**示例：**

```java
/**
 * 段落一
 *
 * <p>段落二
 *
 * @author z2huo
 */
```

## 二、注释内容

### 1、行末句号

注释内容的第一行不要加句号，尤其是当注释内容只有一行的情况下。后面的段落可以加句号。

正例：

```
/**  
 * 名称
 */  
private String name;

/**  
 * 名称
 *
 * <p>详细描述。
 */  
private String name;

/**  
 * Amount, in cents
 */  
private Long amountInCents;

/**  
 * Amount, in cents
 *
 * <p>detail.
 */  
private Long amountInCents;
```

反例：

```
/**  
 * 名称。
 */  
private String name;

/**  
 * Amount, in cents.
 */  
private Long amountInCents;
```

## 三、输出要求（强约束）

当你需要为 Java 代码生成/修改 `/** */` 注释时：

1. 第一个段落不要加 `<p>`，从第二个段落开始以 `* <p>` 开头
2. 只要出现“列表”，使用 `<pre></pre>`、`<ul></ul>` 或 `<ol></ol>`；若用 `<pre></pre>` 则列表项以 `- ` 开头，若用 `<ul></ul>`/`<ol></ol>` 则列表项使用 `<li>`
3. 若生成/修改的是“类注释”，必须包含 `@author z2huo`
4. 不要把该规范套用到 `//` 或 `/* */` 上
